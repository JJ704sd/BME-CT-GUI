## 下一轮行动规划

### 问题 1：FLARE22 taxonomy 错位导致 Dice 无意义

**现状：** `taxonomy_match: True` 只检查了 ID 集合交集，未做语义级匹配。FLARE22 和 AMOS22 的 label ID 含义完全不同。

**行动：**
- 实现自动 taxonomy remap：在后端 `validate_against_custom_label()` 中，当检测到标签 ID 语义不匹配时，自动按器官名重映射
- 或在前端提供手动映射表，让用户指定 FLARE22→AMOS22 的 ID 对应关系
- 映射完成后重新计算 Dice，预期结果应接近离线 remap 的 `mean_dice=0.893`

### 问题 2：AMOS 0117 推理耗时过长（1054s）

**现状：** AMOS 568 层切片在 RTX 4060 Laptop (8GB) 上需要 ~17 分钟。

**行动：**
- 考虑 `fast` profile 作为默认预览模式（耗时 ~384s）
- 调查 `tile_step_size` 对耗时和质量的影响
- 确认 persistent worker 是否能改善连续推理

### 问题 3：标签文件传输 bug 已修复但根因未完全确认

**现状：** 通过新增拖拽支持和调试日志，标签文件现在能正确传输。但原始按钮路径的 bug 未能复现。

**行动：**
- 保留 `console.log` 调试日志一段时间，观察是否再次出现
- 考虑在推理结果回填时，自动从 job 目录读取 validation 结果并展示在 GUI 中

### 问题 4：GUI 评估面板对跨数据集结果的展示

**现状：** 当 taxonomy 错位时，GUI 显示极低 Dice，用户可能误以为模型质量差。

**行动：**
- 当 `taxonomy_match: false` 或 Dice 异常低时，在评估面板显示"标签 ID 不匹配，需 remap"提示
- 提供快速 remap 入口或建议用户使用离线工具
