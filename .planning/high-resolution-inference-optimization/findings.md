# 高分辨率 CT 推理优化发现

## 2026-05-31：高分辨率输入导致推理时间显著延长

**事实：** AMOS CT（768×768×103）在线推理已完成，job `ad3d14eba3de` 使用 `fast` profile 得到 `mean_dice=0.77724`。该输入明显大于标准 AMOS 0117（512×512×N），推理时间显著长于常规病例。

**证据：**

- 输入分辨率：768×768×103
- 标准 AMOS 分辨率：512×512×N
- 面积比：768×768 / 512×512 ≈ 2.25 倍
- 2D 模型逐切片处理：103 层 × 每层 768×768

**判断：** 推理速度下降是预期行为，非系统故障。主要瓶颈是输入分辨率高于模型训练时的标准尺寸；`mean_dice=0.77724` 符合 fast profile 预览定位，不能替代 AMOS `quality` profile 的正式质量基线。

## 2026-05-31：GPU 资源接近上限

**事实：** RTX 4060 Laptop GPU 在推理过程中：

- 利用率：100%
- 显存占用：95%（约 7.6GB / 8GB）
- 温度：57°C
- 功率：27W / 40W（受限于笔记本散热）
- 时钟频率：2505 / 3105 MHz

**判断：** GPU 资源已充分利用，但受限于笔记本散热设计，无法达到满功率运行。升级硬件或使用台式机 GPU 可能改善性能。

## 2026-05-31：2D 模型处理效率

**事实：** 当前使用 `nnUNetTrainer__nnUNetPlans__2d` 模型，将 3D CT 数据按切片逐层处理。

**判断：** 2D 模型在处理高分辨率切片时效率较低，因为：

1. 无法利用 3D 上下文信息
2. 每层需要独立推理
3. 滑动窗口步骤更多

使用 3D 模型可能更高效，但需要更多显存。

## 2026-05-31：重采样开销

**事实：** nnUNetv2 在推理前需要将输入重采样到模型的 patch size（640×640）。

**判断：** 重采样开销相对较小，不是主要瓶颈。主要计算量仍在模型推理阶段。

## 需要继续确认的证据

1. 降采样后的推理时间和质量对比
2. 3D 模型在高分辨率输入上的表现
3. 不同 GPU 硬件的性能差异
4. 高分辨率病例是否需要单独的推荐 profile 或 UI 提示

## 优化可行性评估

| 优化方案 | 可行性 | 预期效果 | 实现难度 |
|---|---|---|---|
| 预降采样 | 高 | 可能显著缩短推理时间，收益需用对照病例验证 | 低 |
| 使用 3D 模型 | 中 | 可能更高效，但需更多显存 | 中 |
| 模型架构优化 | 低 | 需要重新训练 | 高 |
| 升级 GPU 硬件 | 中 | 显著改善性能 | 中 |

## 下一步建议

1. **短期**：实现预降采样选项，让用户选择是否降采样
2. **中期**：评估 3D 模型在高分辨率输入上的表现
3. **长期**：考虑训练支持更高分辨率的模型
