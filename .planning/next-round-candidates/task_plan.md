# 下一轮候选任务规划

**Scope:** 基于 2026-05-28 项目现状，规划下一轮可执行任务

**当前状态：** 三大目标 ~90-95% 完成，核心功能稳定，测试全部通过

---

## 候选任务

### 1. 远程推理分体部署实施

**优先级：** 高

**前置文档：** `.planning/deployment-preparation/task_plan.md`（已完成规划）

**目标：** 本地 Windows 前端 + 远程 Linux GPU 后端，实现"本地导入 CT → 远程推理 → 本地查看结果"

**关键步骤：**
1. 远程 Linux 服务器部署后端（Ubuntu + CUDA + nnUNetv2 环境）
2. `server/main.py` 路径适配（Windows → Linux 跨平台方案）
3. 本地前端 `API_ENDPOINT` 改为远程服务器地址
4. 后端添加 CORS 中间件
5. 端到端验证：SSE 实时进度 + 结果下载

**风险：** 需要可用的 Linux GPU 服务器；SSE 跨域需要正确配置

---

### 2. UI 精细化与交互优化

**优先级：** 中

**目标：** 基于窗预设联动功能的后续优化

**候选改进：**
- 高亮动画持续时间可调（当前 2.2s，用户可能希望更长或更短）
- 预设按钮增加图标或颜色区分
- 器官分类卡片增加点击展开详情
- 底部 progress rail 增加推理模式标签（quality/fast）
- 三视图工具栏增加窗宽窗位数值显示

**风险：** 低，纯前端改动

---

### 3. 多模型支持准备

**优先级：** 低（等待新 checkpoint）

**目标：** 当前只有 AMOS22 单模型，为未来多模型切换做架构准备

**候选改进：**
- `modelOptions` 从硬编码改为从后端 `/api/models` 动态获取
- 后端增加模型目录扫描和模型信息 API
- 前端模型卡片改为可点击切换（当前为静态展示）
- 肺窗/骨窗预设在对应模型可用时自动启用器官映射

**风险：** 需要新 checkpoint 才能验证；当前单模型场景下优先级不高

---

### 4. 推理性能优化

**优先级：** 中

**目标：** 减少推理等待时间，提升用户体验

**候选改进：**
- 持久 worker 模式稳定性和错误恢复
- 推理进度更细粒度（当前只有阶段级百分比）
- 大体积 CT 的预处理优化（多线程 + 进度反馈）
- 缓存命中时的即时结果返回优化

**风险：** 涉及后端推理逻辑，需要充分测试
