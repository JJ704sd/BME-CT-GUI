# 影像量化分析调研结论

## 已确认的结论

1. 当前阶段最适合做的是**基于分割 mask 的器官级量化**。
   - 原因：项目已经有腹部 CT 导入、nnUNetv2 分割结果回填、标签验证和报告导出能力。
   - 量化可以直接复用这些结果，不需要改后端推理链路。

2. 第一阶段应采用**纯前端 CPU 方案**。
   - 原因：量化的核心计算只是对 mask 做一次遍历，复杂度可控。
   - 前端能独立完成，不影响后端 SSE、缓存、validation 和 job summary。

3. 当前可稳定输出的指标是：
   - 体素数
   - 体积
   - 最大轴向截面积
   - 包围盒尺寸
   - 头足向长度估算
   - 三维最长径估算

4. 当前不应伪造成已完成临床级指标的内容是：
   - 壁厚
   - 精确管腔面积
   - 中心线长度

## 为什么选择这些指标

### 1. 体素数

体素数是最直接的分割结果统计量，和 label 的出现范围一一对应，几乎没有解释歧义。

### 2. 体积

体积可以由体素数和 NIfTI spacing 精确换算，是当前最可靠、最容易验证的物理量化指标。

### 3. 最大轴向截面积

赛题明确提到管腔截面积、长度等指标。当前虽然没有壁/腔分离标签，但可以先输出“最大轴向截面积”作为可解释近似。

### 4. 包围盒尺寸与长度估算

这类指标不需要额外模型，只依赖 mask 的空间边界，适合第一阶段快速落地。

### 5. 三维最长径

这可以作为整体器官大小的快速估算，适合报告中作为补充指标。

## 哪些内容当前不做

### 1. 壁厚

不做壁厚数值输出。原因是当前标签体系通常只给整体器官，不区分壁和腔；如果硬算，临床意义不可靠。

### 2. 精确管腔面积

不对胃、肠、胆囊、膀胱等空腔器官输出“精确管腔面积”。当前只能输出整体截面积估算。

### 3. 中心线长度

不在第一阶段实现。中心线提取和骨架化的实现与验证成本都更高，适合后续增强。

### 4. 后端持久化

不修改后端 job summary 和 validation 流程。这样可以降低风险，也避免影响已有推理质量基线。

## 当前量化结果的解释边界

- 体积：可作为可靠物理量使用。
- 截面积：当前是最大轴向面积或近似管腔面积，不等于精确解剖测量。
- 长度：当前是包围盒或头足向范围估算，不等于中心线长度。
- 壁厚：当前版本不输出数值，只提示标签体系不支持。

## 后续优化方向

1. **连通域清理**
   - 去掉小孤岛，减少体积和包围盒受假阳性影响。

2. **PCA 主轴长度**
   - 比单纯 z 向包围盒长度更适合倾斜器官。

3. **中心线长度**
   - 适合食管、十二指肠、血管等管状结构。

4. **更精确的最大径**
   - 可从包围盒最长边升级到表面点/凸包近似的三维最大径。

5. **真正的壁/腔标签**
   - 如果后续新增专用标签或额外模型，才能可靠输出壁厚和精确管腔指标。

## 对报告文案的要求

报告里必须保留以下边界表达：

- 体积、截面积和长度来自自动分割 mask 与 NIfTI spacing 的估算。
- 壁厚和精确管腔指标需要专用标签或后续算法。
- 对主动脉、下腔静脉可以写“管腔截面积近似”。
- 对胃肠、胆囊、膀胱等应写“整体器官截面积估算”。

## 结论

当前方案的目标不是一次性做出全部临床级指标，而是先把**可靠、可解释、可验证**的量化指标接入现有前端和标准化报告流程，再逐步向更复杂的壁厚、中心线和管腔精确测量扩展。
